# 常见问题

## 验证码相关

### 验证码识别失败怎么办？

检查 `data/pages/` 下的调试截图，确认验证码类型。

- **推荐**：切换到大模型识别 `CAPTCHA_SOLVER=llm`，识别率更高，支持点选 + 滑块。详见 [大模型验证码接入指南](LLM_CAPTCHA.md)
- **备选**：使用 `LOGIN_METHOD=qrcode` 扫码登录，完全绕过验证码
- 国网每天有登录次数限制，频繁登录会触发封控，请勿短时间内多次重试

### RK001 网络连接超时

国网检测到异常登录频率，等待几小时后重试。

## 传感器相关

### 阶梯用电传感器无数据

仅住宅用户（户名含「住宅」）有阶梯数据，充电桩等非住宅户号会自动跳过。

### 分时电量数据为空

当月谷/平/峰/尖传感器依赖数据库从 `daily_usage` 表 SQL 汇总：

- 程序需要先写入日用电数据，再汇总出分时电量；首次同步后即可正常显示
- 部分省份日数据可能不含分时字段

### 应交金额始终为 0

如果账户余额充足、无欠费，应交金额为 0 是正常的。Web 控制台会显示"已交清"提示。

## 数据库相关

### 数据库选哪个？

- **SQLite**（默认）：无需额外安装，适合单机 Docker 部署
- **MySQL / PostgreSQL**：适合多实例共享或已有数据库服务的场景

详见 [数据库说明](DATABASE.md)

## Home Assistant 相关

### HA Add-on 启动报 Duplicate mount point

卸载重装 Add-on。

### Web 控制台无法在 HA 中加载（空白/拒绝连接）

如果 HA 通过 HTTPS 访问，而 Web 控制台是 HTTP，浏览器会阻止混合内容。解决方案是配置 Nginx 反向代理将 Web 控制台也转为 HTTPS。详见 [面板集成指南](HA_PANEL.md)。

### MQTT Discovery 实体未自动创建

1. 确认 HA 中已添加 MQTT 集成（设置 → 设备与服务 → 添加集成 → MQTT）
2. 检查 `MQTT_HOST`、`MQTT_PORT` 配置是否正确
3. 使用 MQTT Explorer 等工具确认消息是否发送成功
4. 详见 [MQTT 测试指南](MQTT_TEST.md)

## 部署相关

### Docker 镜像拉取慢

README 中提供了 GHCR 和 Docker Hub 的国内加速镜像地址，优先使用加速源。

### 登录频繁被封控

- 默认每天同步 2 次（设定时间 + 12 小时后），已足够日常使用
- 实际执行时间会在设定值 ±10 分钟内随机偏移，避免固定时间点请求
- 不要设置 `RUN_ON_STARTUP=true` 后反复重启容器
